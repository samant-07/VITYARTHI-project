Problem statement

Typical characteristics of such manual or fragmented bank management processes are those critical inefficiencies in things, such as inaccuracy of data due to human error, waiting time for customers, poor security of data, and inability to present timely, substantive reports.

Project Scope

Key Functional Scope:

Account Life Cycle Management- Complete functionalities of account creation and deletion.

Transaction Processing: Support two major update operations, namely, deposit and withdrawal of funds.

Data Retrieval: Ability to query for and display account details with current balance.

Target user

Students/Learners: Those learning the Python language or software development generally, interested in understanding modular design and how to handle data in a program.

Instructors/Evaluators: This is for quick demonstration and evaluation of the following basic programming concepts: modularity, error handling, basic I/O.

High-level features

Modular Design: Different operations are separated into different packages. The packages used include Create_account, Deposit_amount, Withdraw_amount, Display_account, and Delete_account.

Simple Interactive Menu for easy navigation through main bank operations.

Data Integrity Checks: Deposits are tested for positive amounts and withdrawals for sufficient balance. Simple Interactive Menu: for easy navigation through main bank operations.

Data Integrity Checks: Deposits are tested for positive amounts and withdrawals for sufficient balance. File Persistence: Storage of data so that the details of the account are maintained between different launches of the program.
